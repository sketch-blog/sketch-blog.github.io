ブログにタグクラウド設置

2019/09/02 in #pelican

目次

Pelicanでつくられているこのブログにタグクラウドのプラグインを導入した話。

タグかカテゴリか。

このブログはスタティックサイトジェネレータのPelicanで作られているんですが、ちょっとカテゴリにするか、タグにするかどちらか悩んだのですが、結局タグを使用することにしました。

カテゴリだと1記事に対して、1個しか作れないけど、タグなら複数つけられて、執筆の際に悩まなくてすみますからね。

両方使うとなんかややこしくなるのでしばらくタグのみでやってみます。

それでサイドバーにpelican-plugins/tag_cloud at master · getpelican/pelican-pluginsを導入したときのメモを残します。

プラグインを導入

  • pelican-plugins/tag_cloud at master · getpelican/pelican-pluginsからダウンロードしてPLUGIN_PATHに設置。

  • pelicanconf.pyに

PLUGINS = ['tag_cloud']



TAG_CLOUD_STEPS = 4 #サイズを変化させる段階
TAG_CLOUD_MAX_ITEMS = 100 #最大何個表示するか
TAG_CLOUD_SORTING = 'random' # 並び順　random, alphabetically, alphabetically-rev, size and size-rev
TAG_CLOUD_BADGE = True #記事数のバッジ
  • templateに任意の場所（私の場合はbase.htmlのサイドバーの部分）に
<ul class="tagcloud">
    {% for tag in tag_cloud %}
        <li class="tag-{{ tag.1 }}">
            <a href="{{ SITEURL }}/{{ tag.0.url }}">
            {{ tag.0 }}
                {% if TAG_CLOUD_BADGE %}
                    <span class="badge">{{ tag.2 }}</span>
                {% endif %}
            </a>
        </li>
    {% endfor %}
</ul>
  • スタイルシートに
ul.tagcloud {
  list-style: none;
    padding: 0;
}

ul.tagcloud li {
    display: inline-block;
}

li.tag-1 {
    font-size: 150%;
}

li.tag-2 {
    font-size: 120%;
}

私はカテゴリ風に使うので大きさは変化させていません。

  • タグのスラッグを設定

日本語のタグの場合Release history — Pelican 4.1.1 documentation にあるように、タグのスラッグをpelicanconf.pyで設定します。

TAG_SUBSTITUTIONS = [
    (unidecode('テスト'), 'test', True),
    (unidecode('投稿'), 'post', True),
    (unidecode('お知らせ'), 'infomation', True)

]

こんな感じ。
